# SendSprint C4 Map

These diagrams complete the architecture map used before GitHub issue work.
They describe SendSprint at three C4 levels:

- [System Context](system-context.md): operators, external systems, and package consumers.
- [Containers](containers.md): CLI, API, dashboard, orchestration core, and persistence boundaries.
- [Components](components.md): internal module responsibilities inside the Python package.

Keep this map aligned with [ARCHITECTURE.md](../../ARCHITECTURE.md) and
[docs/architecture/overview.md](../architecture/overview.md) when changing
runtime shape, delivery flow, side-effect policy, or integrations.
